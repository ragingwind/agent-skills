### `/upload-evidence [--pr <number>] [--mode screenshot|video]` — Upload Evidence to PR

Upload browser test evidence (screenshots or video) to the `test-evidence` GitHub Release and post as a PR comment with asset links.

```
Skill(skill: "upload-evidence")
Skill(skill: "upload-evidence", args: "--pr 7")
Skill(skill: "upload-evidence", args: "--mode screenshot")
Skill(skill: "upload-evidence", args: "--mode video")
```

#### When to Use

- After UI tests produce evidence (screenshots or video from agent-browser, Playwright)
- Called by finalize stage (orchestrator), or independently when evidence needs re-uploading

#### Evidence Modes

| Mode | Format | Source location | File pattern |
|------|--------|----------------|-------------|
| `screenshot` (default) | `.png` | `$STATE_DIR/evidence/` | `s<N>-step<NN>-<desc>.<hash8>.png` |
| `video` | `.webm` | `$STATE_DIR/evidence/` | `s<N>-<slug>.<hash8>.webm` |

Auto-detection: if `--mode` is not specified, the script checks `$STATE_DIR/evidence/*.png` first (screenshot mode), then `$STATE_DIR/evidence/*.webm` (video mode). Filenames are content-addressed after `store_evidence_migrate` runs — see `rules/testing.md#file-naming-authority`.

Upload renames to PR-scoped: `pr{N}-{basename}.{ext}`

#### Execution

Run the script — all hard rules are enforced by code:

```bash
bash ${CLAUDE_PLUGIN_ROOT}/skills/upload-evidence/upload-evidence.sh
bash ${CLAUDE_PLUGIN_ROOT}/skills/upload-evidence/upload-evidence.sh --pr 7
bash ${CLAUDE_PLUGIN_ROOT}/skills/upload-evidence/upload-evidence.sh --mode screenshot
bash ${CLAUDE_PLUGIN_ROOT}/skills/upload-evidence/upload-evidence.sh --mode video --name mobile-nav-overlap
```

See [upload-evidence.sh](upload-evidence.sh) for implementation.

#### Hard Rules (enforced by script, not documentation)

These rules are implemented in `upload-evidence.sh` and cannot be bypassed:

1. **Asset link markdown**: `[flow](url)` — generated by Python, never by shell string concatenation
2. **Comment posting**: `--input <json-file>` — never `--field body="..."` (zsh `!` → `\!` breaks syntax)
3. **PR-scoped filename**: `pr{N}-{basename}.{ext}` — derived from source filename, never manual
4. **Release tag**: `test-evidence` — hardcoded, never configurable
5. **Upload verification**: every asset verified via `gh api` before posting comment
6. **Screenshot grouping**: Python groups `.png` by scenario prefix (`s<N>`) in the PR comment
7. **Never silent skip**: script exits non-zero if no evidence found or upload fails
